Manual Testing + Automation Together

How both disciplines worked on the same projects

Manual Testing

Day-to-Day Execution

Every sprint I received User Stories with Acceptance Criteria from the PM. My job: write test cases in Excel (happy path + negatives + edge cases), execute on staging, report bugs in GitLab, retest fixes, deliver a sprint QA summary to Senior QA Lead.

  • Requirement chain: Stakeholder → PM → BRD → Sprint User Story → Acceptance Criteria → my Excel test cases
  • AMRUT example: Zone Board Alert — 1 User Story → 8 test cases covering 5 ACs + AI-brainstormed edge cases (simultaneous over-pressure + trip state; zone with zero assigned stations)
  • IMS example: Inward Stock Form — 1 User Story → 11 test cases including AJAX empty-array state (TC_IN_007 — AI-suggested, validated by me)
  • Every GitLab bug report: Title · Steps · Expected · Actual · Environment · Severity + screenshot
  • Security bug found: View-Only users could see SCADA control buttons — RBAC testing caught it
  • Data bug found: Negative stock at qty = available + 1 — boundary value analysis
  • Agile ceremonies: Planning, standups, demo prep (pass/fail counts, bugs raised vs closed), retrospectives
JS at work · TS for personal growth

Automation Skills

At work I write JavaScript inside the Senior QA Lead's framework. For personal projects I build TypeScript frameworks from scratch — deliberately growing both skill sets.

  • Work (JavaScript): 160+ scripts, strict MR process — Senior reviewed every line before merge, taught me patterns via code comments
  • Personal (TypeScript): DemoBlaze (typed POM + Allure + GitHub Pages), SauceDemo (cross-browser: Chromium/Firefox/WebKit), ReqRes API (typed client, multi-env config)
  • Hardest work challenge — AJAX cascade: IMS inward form needed waitForResponse to intercept subcategory API call before asserting child dropdown — taught by Senior QA via MR comment
  • SCADA real-time challenge: Live sensor data changed every 15s — designed insert-known-value protocol with developer for stable assertions
  • Flaky test resolution: Playwright trace viewer + Cursor AI to diagnose race condition in sensor polling — fixed to zero failures
  • MR learning loop: Senior's comments ("use waitForResponse here", "move locator to page class", "externalise this to JSON") shaped how I write automation today
